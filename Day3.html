The boilerplate code is activated by including the necessary HTML structure and elements in your document. This typically involves:

  1. First of All press Shift + !
  2. Defining the document type with < !DOCTYPE html >
  3. Creating the < html > element with the appropriate language attribute
  4. Including the < head > section with meta tags and the document title
  5. Adding a < body > section where the main content will reside

How Fill path works

The fill path in HTML is used to define the color and pattern that fills a shape or area within an SVG (Scalable Vector Graphics) element. It is specified using the "fill" attribute within SVG elements such as < rect >, < circle >, < ellipse >, < polygon >, and < path >. The fill attribute can take various values, including color names (e.g., "red", "blue"), hexadecimal color codes (e.g., "#FF0000"), RGB values (e.g., "rgb(255, 0, 0)"), or even gradients and patterns defined within the SVG.

How div tag worked

The < div > tag in HTML is a block-level element that is used to group other HTML elements together. It is often used for layout purposes and can be styled using CSS. The < div > tag does not have any semantic meaning on its own, but it can be given a class or id attribute to apply specific styles or scripts to the grouped elements.

how Class tag worked

The class attribute in HTML is used to specify a class for an HTML element. It can be used to apply CSS styles to multiple elements at once or to target specific elements with JavaScript. The class attribute can take multiple class names, separated by spaces, allowing for greater flexibility in styling and scripting.

How ID tag worked

The id attribute in HTML is used to assign a unique identifier to an HTML element. This identifier can be used to target the element with CSS for styling or with JavaScript for manipulation. Unlike class names, which can be shared among multiple elements, an id must be unique within a page.
